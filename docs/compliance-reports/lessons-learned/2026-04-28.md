## Patterns Observed

Because this is the first repo audited in the session, pattern counts refer to occurrences within this single repo rather than cross-repo frequencies. Patterns are flagged for manifest candidacy if they are structurally generic (i.e., they would apply equally to any Python project generated from this or a similar template) or if they represent a class of problem likely to recur.

### Dependency hygiene cluster (4 candidates)

FOUND-101, FOUND-102, FOUND-103, and FOUND-109 are all dependency configuration mismatches against `packages.md`:

- `safety` alongside `pip-audit` (redundant -- packages.md names pip-audit as canonical)
- `python-dotenv` alongside `pydantic-settings` (pydantic-settings subsumes dotenv loading)
- `ruamel-yaml` instead of `pyyaml` (packages.md canonical preference)
- `renovate.json` `enabledManagers` lists `poetry` rather than `uv` (wrong package manager)

These four share the same root cause: the cookiecutter template was written before packages.md was stabilized, and the mismatches propagated forward. Any project generated from this template before the packages.md freeze will carry all four.

### Frontend quality gaps cluster (3 candidates)

FOUND-104, FOUND-105, and FOUND-106 form a consistent gap: the frontend scaffold was never hardened past initial generation.

- Cookiecutter placeholder text visible in `frontend/src/App.tsx` (user-visible)
- No enforced coverage thresholds in Vitest config
- No a11y ESLint plugin (`eslint-plugin-jsx-a11y` or equivalent)

### API design gaps cluster (3 candidates)

FOUND-107, FOUND-108, and FOUND-113 all concern the API layer:

- Entity endpoints lack authentication (deferred Phase 1, but no RAD `#CRITICAL` marker)
- CORS middleware uses wildcard `allow_methods` / `allow_headers`
- EIN value exposed verbatim in 409 response detail strings

### Documentation gaps cluster (3 candidates)

FOUND-110, FOUND-111, and the `docs/architecture/` gap (FOUND-012, already remediated) form a documentation gap pattern:

- `__PROJECT_CREATION_DATE__` placeholder unfilled in `template_feedback.md`
- `docs/response-aware-development.md` referenced in CLAUDE.md but absent
- (FOUND-012 remediated in-session: `docs/architecture/` directory created)

### Single-occurrence candidates

- FOUND-100: `datetime.UTC` used in 3 source files -- Python 3.12 project targets 3.12, but global template targets 3.10+. Requires manifest check scoped by `requires-python`.
- FOUND-112: `tests/integration/` empty -- CI integration flag implies integration tests exist.
- FOUND-114: `utils/financial.py` is a one-line stub -- declared public surface with no implementation.

### Cross-domain recurrence within this repo

- **Mutable references**: PC-012 (pre-commit rev fields) and CI-005-A/B/C (action refs) are the same failure class expressed in two different config files. The manifest already covers these as separate checks, but remediation agents should be paired -- fixing one without the other leaves the project in a partial state.
- **Org reusable workflow opacity**: MKDOCS-007 and SCORECARD:scorecard-workflow-failing both blocked on an org-level reusable workflow that could not be inspected locally. This is a structural audit gap: org-scoped workflows need a separate inspection path.
- **CLAUDE.md as living config**: CLAUDE-002/003/008 all represent CLAUDE.md drift from the global standard. This will recur in every project that was generated before the global CLAUDE.md v1.4.0 baseline.

## Proposed Manifest Additions

The following candidates are generic enough to apply to any Python project from this template family. They are formatted as ready-to-paste YAML entries following the check schema observed in domain agent findings.

### Dependency hygiene checks

```yaml
- id: DEP-001
  domain: dependency-hygiene
  severity: suggested
  description: "safety present alongside pip-audit; packages.md names pip-audit as canonical"
  verify: "not_in_file: pyproject.toml, pattern: 'safety'"
  override_eligible: true
  note: "Remove safety and rely solely on pip-audit. Override if safety is required by an external compliance requirement."

- id: DEP-002
  domain: dependency-hygiene
  severity: suggested
  description: "python-dotenv present alongside pydantic-settings; pydantic-settings subsumes dotenv loading"
  verify: "not_in_file: pyproject.toml, pattern: 'python-dotenv'"
  override_eligible: true
  note: "Use pydantic-settings model_config with env_file instead."

- id: DEP-003
  domain: dependency-hygiene
  severity: suggested
  description: "ruamel-yaml present; packages.md canonical preference is pyyaml"
  verify: "not_in_file: pyproject.toml, pattern: 'ruamel-yaml'"
  override_eligible: true
  note: "Replace with pyyaml unless ruamel-yaml round-trip preservation is explicitly required."

- id: DEP-004
  domain: dependency-hygiene
  severity: important
  description: "renovate.json enabledManagers includes 'poetry' but project uses uv"
  verify: "not_in_file: renovate.json, pattern: '\"poetry\"'"
  override_eligible: false
  note: "Replace 'poetry' with 'uv' in enabledManagers. Incorrect manager causes Renovate to miss uv.lock updates."
```

## Agent Scope Expansion Candidates

**python-toolchain-auditor**: Consider adding a check for `datetime.UTC` usage when `requires-python` is below `>=3.11`. `datetime.UTC` was added in Python 3.11; projects targeting 3.10 must use `datetime.timezone.utc`. The auditor currently checks for tool presence and configuration but does not inspect source files for compatibility-sensitive stdlib calls.

**ci-compliance-auditor**: Consider adding a check that validates `enabledManagers` in `renovate.json` matches the project's declared package manager (inferred from presence of `uv.lock` vs `poetry.lock`). The current CI domain focuses on workflow files; Renovate config lives adjacent to CI concerns but is not currently covered.

**foundations-auditor**: Consider adding a check that `tests/integration/` is non-empty when any CI workflow references an integration test step. The current FOUND-012 check only verifies that `docs/architecture/` exists; a parallel check for test directory population would catch the FOUND-112 pattern systematically.

**claude-config-auditor**: Consider adding a check that all file paths referenced in CLAUDE.md `see <path>` cross-references resolve to existing files. The CLAUDE-003/DOC-002 pattern (reference to a non-existent `docs/response-aware-development.md`) would be caught automatically.

## Priority Action Items Not Resolved This Pass

Listed in priority order. Each item includes the blocking reason and suggested next step.

1. **OSSF-001: Apply for OpenSSF Best Practices badge** (Critical, manual)
   Requires submitting the project URL at https://www.bestpractices.dev/en. No automated fix is possible. Assign to project maintainer. Target: before first release.

2. **SCORECARD:scorecard-workflow-failing** (Critical, blocked on org)
   The org-level reusable workflow referenced by `.github/workflows/scorecard.yml` is not accessible for local inspection. Requires org admin to verify the `ByronWilliamsCPA/.github` repo contains the expected reusable workflow and that the `GITHUB_TOKEN` scope covers it. Target: next sprint.

3. **FOUND-107 / API-001: Unauthenticated entity endpoints** (Warning, phase-deferred)
   Authentication is explicitly deferred to Phase 1. However, the deferral must be tagged with a RAD `#CRITICAL` marker and a phase reference in `entities.py`. The check will then pass on the intent rather than the implementation. Target: before any non-localhost deployment.

4. **SCORECARD:Signed-Releases** (Important, no releases yet)
   No releases exist; this check cannot pass until the first tag is pushed. Ensure the publish workflow signs release artifacts before tagging v0.1.0. The `publish-pypi.yml` workflow should be reviewed for Sigstore or GPG signing before first release.

5. **FOUND-112 / TEST-001: Empty integration test directory** (Warning)
   `tests/integration/` is empty. The CI pipeline has an integration-test stage. Add at least one test covering the entity CRUD happy path against a real (or Testcontainers-managed) PostgreSQL instance. Target: Phase 1 alongside authentication.

6. **FOUND-104 / FE-001: Cookiecutter placeholder in App.tsx** (Warning, user-visible)
   The placeholder text is visible in the running application. This is a post-generation cleanup task that should have been caught before first commit. Fix immediately.

7. **FOUND-111 / DOC-002: Missing docs/response-aware-development.md** (Important)
   Referenced in CLAUDE.md line referencing `docs/response-aware-development.md`. Either copy from the global template repo or update the reference. This is a one-file fix.

8. **DEP-004 / FOUND-109: renovate.json enabledManagers lists poetry** (Important)
   Renovate will not process `uv.lock` updates with this configuration. The fix is a one-line change to `renovate.json`. This should be treated as a CI correctness issue, not a style preference.

## Template Feedback Generated This Session

The following items have been added to `docs/template_feedback.md` (or should be added if not already done by a remediation agent):

- **Mutable rev fields in starter pre-commit config** (Critical): Template should ship with pinned SHA revs or include a post-generation hook that pins them.
- **@main action refs in starter workflows** (Critical): Starter workflows should use SHA-pinned action refs from day one.
- **Missing settings.json** (Critical): Template should include a `.claude/settings.json` with at minimum `{"permissions": {"allow": [], "deny": []}}`.
- **poetry in renovate.json** (Important): Template's `renovate.json` should use `uv` in `enabledManagers`, not `poetry`.
- **`__PROJECT_CREATION_DATE__` unfilled** (Suggested): Post-generation hook or cruft configuration should substitute this value automatically.
- **Empty integration test directory** (Suggested): Either populate with a stub test or remove the CI integration stage from the starter workflow.
- **CORS wildcard in starter main.py** (Suggested): Starter should enumerate specific methods rather than using wildcards.
