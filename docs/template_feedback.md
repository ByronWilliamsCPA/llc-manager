### `python-compatibility.yml` defaults `--no-build` to true for hatchling projects

- **Priority**: High
- **Category**: CI/CD
- **Discovered**: 2026-05-16

**Issue**: The `.github/workflows/python-compatibility.yml` caller generated by the template does not set `no-build: false`, so the upstream reusable workflow at `ByronWilliamsCPA/.github/.github/workflows/python-compatibility.yml` runs `uv sync --all-extras --frozen --no-build`. For projects that use the `hatchling` build backend (which this template selects in `pyproject.toml`), uv refuses to install the local editable distribution and fails with `Distribution ... can't be installed because it is marked as --no-build but has no binary distribution`. Every matrix combination then fails identically.

**Context**: Discovered while landing the first PR that actually changed `src/` files in this project (path filter on `python-compatibility.yml` only triggers on Python file changes). All six matrix jobs (Python 3.12/3.13 × Ubuntu/macOS/Windows) failed at the dependency-install step.

**Suggested Fix**: In the generated `.github/workflows/python-compatibility.yml`, set `no-build: false` in the `with:` block (the upstream workflow's own input description literally says "disable for projects with a build backend like hatchling"). Same applies to the `python-ci.yml` caller in `.github/workflows/ci.yml` — it happens to currently succeed (likely via cache), but is similarly fragile.
